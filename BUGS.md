## 2026-05-20 — 曲线笔触仍然「断」（v0.2 的补丁不够彻底）
- **症状**：v0.2 的 round-join 补丁理论上盖住外侧楔形 gap，但用户实际仍能看见「稀缝」；笔触越粗 / 弯越急越明显
- **根因**：v0.2 是「每段矩形 stamp + cur 处补丁」，相邻段在公共端点用**不同方向**的短边相接 → 切割线那一行像素**两端列 alpha < 100%**（抗锯齿羽化），两块 stamp 在公共边叠加时 alpha 50% + 50% = 75%（不是 100%），看起来就是稀缝。补丁也是矩形，源 1px 中行同一行像素 → 同样的 alpha 问题
- **修法**：彻底换算法——用 affine quad strip：相邻段共享 `path[i] ± perp[i]` 端点，几何上严丝合缝（不重叠不留缝），无 alpha 叠加；源 1px 中行抽到独立 `midRow` canvas，避免 affine 采样污染相邻行
- **文件**：[index.html](index.html) line 446（`drawAffineTri`）/ line 479（重写后的 `drawCurvedStretch`）/ line 386（`midRow` 抽取）

## 2026-05-19 — 曲线笔刷转弯处断裂
- **症状**：曲线模式画弧线、转弯，外侧出现明显空白缝；笔触越粗、转弯越急越明显
- **根因**：`drawCurvedStretch` 每段中行用**自段切线**做旋转，相邻段在公共端点旋转方向不同 → 外侧缝隙 ≈ `(Wd/2)·tan(Δθ/2)`，笔触 200px、转弯 30° 时缝隙达 27px。原 `+1px overlap` 完全不够
- **修法**：每个内部 path 点加一片**平分角方向**中行 stamp（圆角接头），高度按 `Wd·tan(|Δθ|/2)` 自适应；同时把采样从 2.5px 缩到 1.5px 减小单段角度差
- **后续**：2026-05-20 发现这个补丁仍有 alpha 叠加问题，v0.3 彻底重写（见上）
- **文件**：[index.html](index.html) line 376（接缝补丁）/ line 445（采样阈值）—— 已被 v0.3 删除
